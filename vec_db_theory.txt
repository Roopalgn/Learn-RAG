# 1. What a Vector Is

* A **vector** is a numerical representation of data in **n-dimensional space**.
* Each number represents a **feature of the data**.
* In AI/ML, vectors convert data into **numbers that algorithms can process**.

Example:

```
[0.25, 0.81, 0.14, 0.92]
```

Key idea:

* Vectors represent **direction and magnitude mathematically**.
* In AI they represent **features of data**.

---

# 2. What an Embedding Is

An **embedding** is a **special type of vector used in machine learning**.

Properties:

* Converts **raw data → vector representation**
* Preserves **semantic meaning and relationships**

Example:

| Word  | Embedding          |
| ----- | ------------------ |
| cat   | [0.22, 0.51, 0.81] |
| kitty | [0.21, 0.52, 0.79] |

Because meanings are similar → vectors are close.

Important rule:

**All embeddings are vectors, but not all vectors are embeddings.**

Embeddings encode:

* meaning
* context
* relationships between items

---

# 3. Why Vector Databases Are Needed

Over **80% of data in the world is unstructured**, including:

* Images
* Audio
* Videos
* PDFs
* Articles
* Text

Traditional databases can store this data but **cannot search it efficiently**.

Vector databases solve this by converting data into **numerical vectors** that allow:

* similarity search
* semantic search
* fast AI retrieval

---

# 4. Core Idea of Vector Databases

Vector databases store **embeddings (vectors)** and retrieve **similar vectors** using distance calculations.

Example:

```
Lake → [0.05, 0.10]
Water → [0.053, 0.11]
```

Since vectors are close → meanings are similar.

This enables **semantic search instead of keyword matching**.

---

# 5. Advantages of Vector Databases

### Efficient Representation

* Converts complex data into **high-dimensional vectors**.

### Similarity Search

* Finds items with **similar meaning**, not just exact matches.

### Scalability

* Uses **vector indexing algorithms** for fast searching.

### Big Data Handling

* Works efficiently with **large datasets**.

### Real-Time AI Applications

* Enables fast responses for AI systems.

# 6. Traditional Databases (RDBMS)

Traditional databases are **Relational Database Management Systems (RDBMS)**.

They store data in **tables**.

Structure:

| CustomerID | Name | Address | Phone |

Components:

* **Rows** → records
* **Columns** → attributes

---

## SQL (Structured Query Language)

Used to:

* retrieve data
* filter data
* join tables
* aggregate data

Example:

```sql
SELECT * FROM Customers WHERE Name = 'John';
```

---

# 7. Schema in Traditional Databases

Traditional databases require a **schema** before data is inserted.

A schema defines:

* tables
* fields
* relationships
* indexes
* constraints

This ensures:

* consistency
* data integrity
* structured organization

---

# 8. ACID Properties

Traditional databases follow **ACID principles**:

* **Atomicity** – transaction fully completes or fails
* **Consistency** – database remains valid
* **Isolation** – transactions don't interfere
* **Durability** – saved data persists

Important for:

* banking systems
* financial systems

---

# 9. Indexing in Databases

Indexes improve **data retrieval speed**.

Instead of scanning the entire table, the database **directly locates the data**.

Without index → **full table scan (slow)**.

---

# 10. Limitations of Traditional Databases

Traditional databases struggle with:

### Unstructured Data

* images
* audio
* video
* text documents

### Scalability

Complex queries on large datasets become slow.

### Flexibility

Schema must be predefined.

Because of these limitations → **vector databases are used for AI systems**.

---

# 11. How Unstructured Data Becomes Vectors

### Step 1 — Raw Data

Unstructured data enters the system:

* documents
* images
* audio
* videos

---

### Step 2 — Data Chunking

Large documents are **split into smaller pieces**.

Example:

Document
→ Chunk 1
→ Chunk 2
→ Chunk 3

Smaller chunks improve:

* search accuracy
* embedding quality

---

### Step 3 — Embedding Creation

Each chunk is converted into a **vector embedding**.

Embedding contains:

* vector representation
* original text (metadata)

Example:

| Vector         | Text             |
| -------------- | ---------------- |
| [0.34,0.91...] | "Cats hunt mice" |

These embeddings are stored in a **vector database**.

---

# 12. Querying a Vector Database

### Step 1 — User Query

User asks a question in natural language.

Example:

> "Tell me about cats"

---

### Step 2 — Query Embedding

The question is converted into a **vector embedding**.

---

### Step 3 — Index Search

Vector databases build an **index** to search vectors efficiently.

The system compares:

* query vector
* stored vectors

---

### Step 4 — Similarity Retrieval

The system finds **closest vectors**.

Example results:

* "Cats hunt mice"
* "Kittens are playful"

---

### Step 5 — LLM Processing

The retrieved information is sent to an **LLM**, which generates the final response.

---

# 13. Vector Database Workflow

Full pipeline:

1. Raw documents
2. Text chunking
3. Create embeddings
4. Store embeddings in vector database
5. User query arrives
6. Query converted to embedding
7. Similarity search
8. Relevant data retrieved
9. Sent to LLM
10. LLM generates response

This process powers **RAG (Retrieval Augmented Generation)** systems.

# 15. Key Algorithms Used

Vector databases use:

* **Nearest Neighbor Search**
* **Approximate Nearest Neighbor (ANN)**

These algorithms find **vectors with smallest distance**.

# 16. What a Large Language Model (LLM) Is

A **Large Language Model** is an AI model trained on **large amounts of text data** to generate human-like responses.

Process:

Input → Model → Output

Example:

Input:

```
Explain AI in one sentence
```

Output:

```
AI is the simulation of human intelligence by machines.
```

### Step 3 — Fine-Tuning

# 18. Transformer Architecture

Transformers help models:

* understand context
* analyze relationships between words

# 19. Self-Attention Mechanism

Self-attention allows the model to examine **relationships between all words in a sentence**.

Example:

"The cat sat on the mat because it was tired."

Self-attention helps the model understand **“it” refers to the cat**.

This allows understanding of:

* context
* semantics
* word relationships
